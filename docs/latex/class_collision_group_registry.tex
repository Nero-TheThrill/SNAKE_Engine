\doxysubsection{Detailed Description}
Manages collision groups using named tags and bitmasks. 

This utility allows mapping string-\/based collision group tags to unique bit values, which can then be used to construct collision layers, filters, or masks.

Useful for implementing collision matrix logic such as "{}\+Player only collides with Enemy"{}.

Retrieves or assigns a unique bitmask for a given collision group tag. 

This function maps a string-\/based collision group name (e.\+g., "{}\+Player"{}, "{}\+Enemy"{}) to a unique bit in a 32-\/bit integer. If the tag is new, the next available bit (1 \texorpdfstring{$<$}{<}\texorpdfstring{$<$}{<} N) is automatically assigned.

This function is not used directly by users. It is called internally by \doxylink{class_object_a7ce1a12c241c5a64ec03e67ee17e1260}{Object\+::\+Set\+Collision()}, which converts user-\/provided tags into bitmask values for collision category and mask.

Returns the collision group tag associated with a given bitmask. 

Performs a reverse lookup from a single-\/bit mask (1 \texorpdfstring{$<$}{<}\texorpdfstring{$<$}{<} N) to the string tag originally registered. This is primarily useful for debugging, logging, or editor UI where human-\/readable names are needed.

If the bit is not registered, an empty string is returned.
