Detailed Description

A spatial hash grid used to accelerate broad-phase collision detection.

SpatialHashGrid divides the 2D world into fixed-size grid cells and maps objects into those cells based on their collider positions.

It reduces the number of collision checks by limiting narrow-phase testing to objects that share or neighbor the same cell.

Used internally by ObjectManager and the collision system.

◆ ComputeCollisions()

void SpatialHashGrid::ComputeCollisions ( std::function< void(Object *, Object *)> onCollision)

Checks for potential collisions between objects within the same grid cells.

Iterates over each populated cell and performs pairwise narrow-phase collision checks between objects in that cell.

If a collision is detected, the provided callback function is invoked with both objects. This approach significantly reduces the number of collision checks versus a full O(n^2) pass.

Parameters
onCollisionCallback function to invoke upon confirmed collision between two objects.
Note
This is the main entry point for broad-phase + narrow-phase collision handling.

void SpatialHashGrid::Insert ( Object * obj)

Inserts an object into the spatial grid based on its collider's position.

Computes the object's grid cell coordinates using its world position, then stores the object pointer in that cell.

The object must have a valid Collider, and its world position should already be synced. This function supports coarse spatial partitioning for broad-phase collision optimization.

Parameters
objPointer to the object to insert.
Note
Called once per object per frame before ComputeCollisions().
